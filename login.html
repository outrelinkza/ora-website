Login & Testing Instructions

Instructions for Google OAuth verification testing

Important: ORA is a mobile application (iOS/Android). The OAuth consent screen is accessed through the mobile app, not through a web login page.

How to Access the OAuth Consent Screen

  1. Install the ORA mobile app on an iOS or Android device
  2. Open the app and complete the initial onboarding (if first time)
  3. Navigate to the "Inbox Shield" tab in the bottom navigation
  4. Tap the "Connect Gmail" button (or similar connection prompt)
  5. The OAuth consent screen will appear - This is the Google OAuth consent screen that requests Gmail read-only access
  6. Sign in with a test Google account (2FA should be disabled for testing)
  7. Review and accept the permissions - The app requests:
    • Gmail read-only access (to scan for brand collaboration emails)
    • User profile information (email and basic profile)

Test Account Requirements

For Testing:

  • • Use a Google account that has been added as a test user in Google Cloud Console
  • • 2FA (Two-Factor Authentication) must be disabled on the test account
  • • The account should have access to Gmail
  • • Test accounts can be added in: Google Cloud Console → APIs & Services → OAuth consent screen → Test users

What the OAuth Flow Does

The OAuth consent screen appears when users want to connect their Gmail account to ORA's "Inbox Shield" feature. This feature allows influencers to:

  • Scan their Gmail inbox for brand collaboration opportunities
  • Identify emails from brands and agencies
  • Track collaboration requests and opportunities
  • Never miss important brand partnership emails

The app only requests read-only access to Gmail, meaning it can read emails but cannot send, delete, or modify them.
